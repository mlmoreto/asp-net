ASP.NET - Breve histório

O ASP.NET é uma tecnologia de desenvolvimento de aplicações WEB, que foi lançada oficialmente em 2002 junto com a plataforma .NET 1.0, nessa época era conhecido como ASP+ por ser o sucessor direto do ASP 3.0 (Clássico).

Foi o primeiro framework de desenvolvimento web da Microsoft.

Mas afinal, o que é framework ?

Um framework captura a funcionalidade comum a várias aplicações. As aplicações devem ter algo razoavelmente grande em comum: pertencer a um mesmo domínio de problema.

Importante lembrar que o ASP.NET roda sob a plataforma .NET, portanto temos todos os benefícios da plataforma.

Evolução do ASP.NET

  • ASP.NET 1.0 - 2002
  • ASP.NET 1.1 - 2003
  • ASP.NET 2.0 - 2005
  • ASP.NET 3.5 - 2007
  • ASP.NET 4.0 - 2010
  • ASP.NET 4.5 - 2011

    • OBS: Não existiu o ASP.NET 3.0, somente plataforma .NET 3.0.

    A web possui características intrínsecas, que não podem ser desconsideradas em hipótese alguma quando se planejam aplicações para serem executadas neste ambiente.

    Visando isso, a Microsoft criou o ASP.NET MVC que é um uma tecnologia (framework) que foi concebida para respeitar as premissas web.

    ASP.NET MVC? Por quê?

    Conforme mencionamos acima, por muitos anos, a Microsoft apostou em um modelo diferente daquele tradicionalmente conhecido por desenvolvedores web (a saber, ASP.NET Webforms). Muito embora esta postura tenha colaborado em grande escala para arrebanhar desenvolvedores de outros universos para o da Internet, isso acabou por gerar alguns "problemas" nas aplicações, dentre os quais citamos:

  • Dificuldade de manutenção;
  • Pouco controle na geração do HTML;
  • Difícil de testar;
  • Preocupação com o ViewState;
  • Não separação de responsabilidades;

    • E eis que passados sete (ou oito) anos do surgimento do ASP.NET, em 2008/2009 o pessoal da Microsoft apresentou o ASP.NET MVC Framework, que habilita a criação de aplicações ASP.NET no famoso e cultuado padrão MVC (Model-View-Controller).


    Curiosidades:


    Web Forms são arquivos com extensão .aspx que surgiram junto com o ASP.NET lá no início, com a versão 1.0 do framework. Falando apenas do mundo Microsoft, os Web Forms surgiram como forma alternativa às páginas .asp, que usam basicamente HTML e VB script.

    As páginas .aspx são escritas em uma linguagem de marcação própria, também chamada de ASP.NET. Um Web Form possui uma tag chamada form no início, com um atributo: runat=’server’. Esse atributo indica que a página é processada no servidor.


    Com qual versão do .NET Framework e do Visual Studio é possível usar o ASP.NET MVC Framework?

    É possível utilizá-lo a partir do .NET Framework 3.5 e Visual Studio 2008, inclusive na versão Express.

    Principais Diferenças entre ASP e ASP.NET:

    ASP

  • Código interpretado
  • Código misturado (Html com VbScript)
  • Linguagem VbScript
  • Menos seguro e menos rápido

    • ASP.NET

  • Código compilado
  • Código separado (ASPX e CodeBehind)
  • Linguagens (Visual C#, Visual Basic)
  • Mais seguro e mais rápido
  • Rico em controles que facilitam criação da página
  • Possue Master Page uma herança visual
